### III. The Build Strategy: How to Start (Hour by Hour)

Since you have 3 people and existing code, split the work like this:

#### **Phase 1: The Skeleton (Hours 0 - 3)**
* **Vibe Coder 1 (The Integrator):**
    * Take your **Python SMS script** and **ML Model**. Wrap them in a simple API (FastAPI or Flask) or a Supabase Edge Function.
    * *Goal:* We need to send a request `get_prediction(user_id)` and get a JSON response `{"range": "12k-15k"}`.
* **Vibe Coder 2 (The UI Builder):**
    * Use **Lovable/Claude** to generate the UI Shell.
    * *Prompt:* "Mobile first React app, bottom navigation bar with 4 items. Home page has a gauge chart. Color scheme is Green #2E7D32."
* **Designer:**
    * Create the **"Bike Filling" Animation/SVG**.
    * Create the **"Agent Avatar"**.

#### **Phase 2: The Brain Transplant (Hours 3 - 7)**
* **Vibe Coder 1 & 2:** Connect the UI to the Logic.
    * **Home Page:** When the page loads, fetch the "Predicted Range" from your ML POC and display it under the meter.
    * **Tracker Page:** Connect the "Simulate Sync" button to your SMS parsing logic. Show the parsed data in the list.
    * **Onboarding:** Port the logic from your Web App POC. If the web app is React, copy-paste the component. If not, ask Claude to "Convert this logic to React."

#### **Phase 3: The "Agentic" Layer (Hours 7 - 10)**
* **All Hands:** Implement the **Voice Button**.
    * Use a simple speech-to-text API.
    * *Hardcode the Demo Path:* If the user says "Bike", the app *must* navigate to the Goal page. If the user says "Income", it *must* add money.
    * *Why:* This satisfies the **"Agentic AI"** requirement in the problem statement.

#### **Phase 4: Polish & Pitch (Hours 10 - 12)**
* **Designer:** Fix the fonts (Use a clean Google Font like 'Poppins' or 'Hind' for Indian context).
* **Business Model Check:** Add a small "Verified by Nischint" badge on the Goal page to hint at the "Freemium/Trust" model.
